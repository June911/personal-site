01定义与边界

SaaS = 订阅制 + 持续交付 + 标准化软件。在讨论"SaaS 是否已死"之前，先要精确定义这三个词圈出的边界——越界的不是 SaaS 在进化，而是变成了另一种东西。

经济边界：边际成本趋零

SaaS 的经济基础是：每多服务一个客户，成本几乎不增加。这使订阅制定价成为可能——固定费用分摊在无限用户上，供应商和客户都受益。

线外反例：Spotify 每增加一个用户就要多付版税，利润结构更像内容分发；AI-native SaaS 每次推理都消耗算力，边际成本回到线性增长。

技术边界：同一份代码服务所有人

标准化的核心是多租户——用同一套系统服务不同客户。一旦为每个客户维护独立分支，就不再是 SaaS，而是外包。

线外反例：定制化外包（Private Branch）、Palantir 早期的驻场部署模式。

功能边界：嵌入客户决策链

订阅制要求客户持续付费，这意味着软件必须成为工作流中不可跳过的节点——不是用完一次就走，而是每天都要经过。

线外反例：格式转换工具、一次性迁移脚本——有用，但不值得订阅。

价值边界：价值锚在软件本身

SaaS 的客户为软件功能付费。如果价值锚不在软件而在其他东西上（社交关系、内容、数据本身），那就是另一种生意披了 SaaS 的外衣。

线外反例：微信（价值锚在社交关系）、Bloomberg Terminal（价值锚在独占数据）。

四条边界的逻辑链
边际成本低
→
才能标准化
→
才能嵌入流程
→
才能形成价值沉淀

这四条边界是层层递进的：没有经济基础就无法标准化，无法标准化就进不了工作流，进不了工作流就没有持续价值。AI 正在从第一条——经济边界——开始动摇整个链条。

02AI 对 SaaS 的冲击

AI 正在从三个层面瓦解传统 SaaS 的构建壁垒。SOP 的供给，从稀缺走向充裕。

构建速度剧变

过去搭建一套行业工作流，需要一群懂行业的工程师干几年。现在，业内人士通过 AI + Skill，1～2 周就能把工作流搭好。

能力门槛骤降

构建能力的载体从「写代码」变成「写自然语言」。人才成本、技术成本都在极速降低。过去靠工程师堆起来的护城河正在被侵蚀。

平台权力迁移

过去 SaaS 之上可以构建一层 App Store（如 Salesforce AppExchange）。未来 Agent 直接对接底层 API / Skill，绕过 SaaS 平台层。权力从「平台」流向「Agent 本身」—— Agent 自己就是 App Store。

本质转变

AI 之前：护城河 = f(构建复杂度) — 能力稀缺，先做到的人赢

AI 之后：护城河 = f(许可稀缺度) — 谁拥有不可替代的约束

Agent 绕过了 SaaS 的界面，直接对接 API。SaaS 曾经承担的「工作流编排」角色，被拆解成了编排层 + Agent + 数据接口三个独立层。

这就像 Google 时代的黄页——大量 SaaS 卖的不是数据本身，而是「把数据变成可用的」这个处理过程。当 LLM 让处理过程的成本趋零，这些 SaaS 就是智能时代的黄页。

04商业模式的困境

Per-seat 定价的瓦解

SaaS 的 per-seat 定价建立在「人是使用单位」这个假设上。当 Agent 成为使用单位，定价模型需要根本重写——按调用次数、按结果、按节省的人力成本计价。

一个按席位收费的公司，客户扩张时收入自然增长；一个按调用收费的公司，效率提升反而可能让收入下降。

边际成本结构变化

传统 SaaS 的边际成本几乎为零。AI 时代，每次推理都需要电力和算力，边际成本变成线性扩张。这对利润率模型是根本性改变。

回到第一节的定义边界——AI-native SaaS 正在逼近经济边界的外侧。当推理成本随用量线性增长，利润结构不再像软件公司，更像内容平台。它们不是 SaaS 在进化，是在变成另一种东西。

自建门槛降低

构建成本大幅降低后，企业面临更现实的「自建 vs. 采购」抉择。以前自建太贵，只能买 SaaS；现在用 Agent + Skill 可能自己就搞定了。竞争从 3 家变 300 家，定价权被碾碎。

核心机制

AI 不是替代软件，是减少使用软件的人。软件还在，但付费的「人头」在消失——per-seat 模型在结构性萎缩，整个 SaaS 行业的估值倍数可能永久性下了一个台阶。

估值陷阱

不需要收入下降就能让股价崩盘——只需估值倍数压缩。一家 15x 营收的公司，当市场认为定价权在消融，可能跌到 6x。收入不变，股价跌 60%。

核心判断

绝大部分 SaaS 公司确实会很惨，但有一部分——因为受到强约束保护——会活得很好。

AI 没有消灭对标准化软件的需求，但彻底改变了供给侧——构建成本趋零后，"标准化"的稀缺性在瓦解。问题不再是谁能做出好软件，而是谁拥有不可替代的约束。

核心假设

SOP 仍然重要：Agent 需要约束（权责分配、偏好、决策授权）来保证执行准确性，SOP 不会消失，只是供给方式变了。

构建能力在贬值，独占数据不会：当能力不再稀缺，护城河从「做得到」转向「拿得到」——许可稀缺取代能力稀缺。

05什么能活下来：数据的独占性

SOP 供给不再稀缺之后，真正决定 SaaS 生死的不是「构建能力」，而是数据的独占性。问一个问题：这份数据离开你，别人能不能拿到同样的东西？

生产独占
数据生产过程是我的，别人物理上拿不到 · 95%
功能独占
数据离开我会降级（失去合规状态/整合上下文）· 70%
惯性锁定
数据在我这里只因为惯性，Agent 可穿透 · 30%

生产独占 — 最硬

不可计算的东西。Bloomberg 的实时交易数据不能爬取、不能合成、不能从第三方授权。在 LLM 世界里，这种数据成为每个 Agent 都需要的稀缺输入，定价权反而增强。

能源同理。AI 用量 ↑ = 耗电 ↑，而发电/输电扩容是年级别的物理工程。数据来源绑定物理过程，这是最硬的独占。

但要区分真独占和伪独占。「我们比别人整理得更好的公开数据」——这不是独占，AI 正在让这类优势快速贬值。真正的生产独占是：数据在你的物理过程中产生，别人连原始材料都拿不到。

功能独占 — 数据离开就降级

数据本身可能属于客户，但离开你的系统就失去了关键属性：合规状态、整合上下文、或认证标记。

Veeva 的 GxP 验证环境就是例子——产品本质上是合规认证本身，不是软件。监管机构认证的是系统，不是功能。数据迁出 = 丧失合规状态。这是目前最坚实的功能独占之一。

但存在一个临界点。AI 输出本身不可完全复现——即使温度设为 0，底层 GPU 的浮点数差异也会导致同一输入产出不同结果。法律如何认定这种不确定性，决定了合规壁垒的命运：

  • 法律不接受幻觉 → 合规壁垒更深更固化，传统 SaaS 反而更安全
  • 法律接受一定容错 → 任何公司的 Agent 都能走合规流程，壁垒消失

关键推论：如果法律能接受判决中的幻觉，为什么不能接受别家 Agent 的合规流程？法律壁垒本质是标准化门槛——一旦 AI 学会这个标准，壁垒就消失。趋势看，法律会慢慢接受 AI 产出，合规壁垒长期可能不可靠。

传统支付轨道也属于功能独占——数据在特定制度框架内才有效力。但制度约束正在被穿透：区块链/稳定币用代码替代信任中介，Agent 天然偏好可编程的链上轨道。这是五年维度里变化最剧烈的子类。

容错率决定功能独占的厚度

准确性要求高 + 数据能持续优化 → 独占性最强。容错率越低的场景，功能独占越接近生产独占——因为重建合规状态的成本趋近于重新生产数据。

惯性锁定 — Agent 可穿透

「信任」听起来很硬，但拆开看，大部分是其他独占层级的伪装：

  • 法律伪装：Basel III 要求银行用「认可的评级机构」。基金经理用 S&P 不是因为信任，是因为监管要求——这其实是功能独占
  • 协调惯性：所有人都用同一标准作为共同语言。Agent 重新收敛到新标准的速度远快于人类
  • 网络效应：如果每个交易对手都用 Bloomberg，你就必须用。这对人类很强。但 Agent 可以同时在所有平台上——不存在「只能选一个」的约束，协调成本趋零

纯惯性锁定只剩不可验证的人类主观判断（医生的临床直觉、投行的并购建议）和尚未被穿透的协调成本。范围在持续缩小。

当前： 生产独占 > 功能独占（合规）> 功能独占（制度）> 惯性锁定 5年后： 生产独占 > 功能独占（合规）>> 惯性锁定 > 功能独占（制度） 变化最大：制度类功能独占（被 Crypto 穿透） 唯一上升：能源类生产独占（物理刚性不变，其他在弱化） 最虚： 惯性锁定（大部分是伪装，纯惯性范围持续缩小）

警惕假壁垒

「数据存在我这里」≠ 壁垒 — 是迁移惯性，AI 在降低迁移成本。Klarna 已证明大客户说走就走。

「迁移很痛苦」≠ 独占 — 竞争对手愿意花时间攻破，壁垒终将被瓦解。

「合规是刚需」≠ 高价值 — 客户当成本买不当投资买，定价天花板有限。

「UI 护城河」在自然语言时代崩塌 — 当界面变成对话，训练用户使用你界面的壁垒就没了。

06评估框架

评估一个 SaaS 公司，核心问题：它的护城河是靠「做得到」还是「拿得到」？

如果是前者（构建能力），LLM 正在摧毁它。如果是后者（许可壁垒），看具体是哪种约束，以及那种约束自身在强化还是弱化。

AI 时代 SaaS 评估清单

  1. 独占性 — 生产独占 > 功能独占 > 惯性锁定
  2. 不可替代性 — 合规 > 增收 > 降本（注意：不可替代 ≠ 高价值）
  3. 数据必要性 — 构建能力 vs 行业知识积累（后者更持久）
  4. 价值捕获 — 按流量/结果计价 > per-seat 定价
  5. 容错率 — 容错率低 + 数据可优化 = 垄断性最强

合规是刚需但定价天花板低，增收类（如 Bloomberg）客户支付意愿最强。不可替代性描述的是「能不能被 AI 替掉」，不是「客户愿意付多少钱」。

数据真正有价值的三个条件

  1. 汇聚型数据 — 单个客户的数据不够，必须跨客户聚合才产生额外价值
  2. 边际收益递增 — 数据越多效果越好，且边际收益不能递减太快
  3. 不可替代数据源 — 数据生产过程依赖我的基础设施，上游不对竞争对手平等开放

Tesla FSD — 物理独占的教科书

传感器在 Tesla 的车上，数据从出生就是 Tesla 的。截至 2026 年 2 月，FSD 累计超 80 亿英里，2026 前 50 天跑了 10 亿英里。完美飞轮：更多车 → 更多数据 → 模型更好 → 更多订阅 → 更多数据。没有人能买到、复制或替代这个数据采集过程——除非你也有几百万辆车在路上跑。

FSD 已转纯订阅（$99/月），约 110 万活跃用户 ≈ $13 亿+ ARR。「用户」是车辆不是人——车辆不会被 AI 替代，Robotaxi 场景下每辆无人车都订阅 FSD = 更多收入。

风险：执行风险巨大——Musk 反复推迟无人驾驶时间线。安全数据方法论被质疑。监管不确定性高。估值已定价完美执行。

Tempus AI — 十年数据管道的壁垒

4,000 万+ 研究记录、150 万+ 临床-基因组匹配、700 万张病理切片、70 亿+ 临床笔记。与 4,500+ 家美国医院建立数据管道，花了 10 年、数十亿美元。关键：数据不属于单个医院，是跨机构聚合后才有价值；上游数据源不对竞争对手开放同等管道。

不是 per-seat 模型。Insights 按数据集付费，Genomics 按检测量计价，算法诊断按使用计价。AI 让药企做更多研究 → 需要更多数据 → 收入增加。Q4 2025 收入 $3.67 亿（+83% YoY）。

风险：盈利能力未证明（2025 年刚转正 EBITDA $500 万），负债 $5.18 亿。隐私法规风险——如果数据共享限制收紧，管道可能受损。

Salesforce — 反面教材

CRM 数据是最标准化、最容易迁移的数据。多存十万条客户记录，产品体验零提升。Klarna 已证明大客户可以说走就走——数据离开 Salesforce 不但零降级，可能更轻便。

自己把客服从 9,000 裁到 5,000。90 个企业账户已观察到 10% 席位减少。Agentforce 越好 → 客户需要越少人 → 席位越少 → 收入越低。三种定价模型并行（$2/对话、$0.10/动作、$125-550/用户/月）= 不知道怎么收钱。

HubSpot 逆势 20% 增长说明 CRM 赛道还在，但 Salesforce 的定位最脆弱——它卖的是「把数据变得可用」，正是智能时代的黄页。

09总结

AI 时代对 SaaS 的冲击是结构性的：SOP 构建成本趋零，per-seat 模式瓦解，边际成本从零变为线性，平台层被 Agent 绕过。

但 SaaS 不会消亡——约束不会消亡。嵌入物理世界的 SaaS（传感器、基础设施）壁垒最硬；嵌入法律和监管体系的，短期安全，但存在临界点——当 AI 系统的合规能力被法律接受，这层壁垒可能反转。

案例验证表明，真正的壁垒来自物理世界而非软件——Tesla 的传感器、Axon 的摄像头、Tempus 的医院管道。数据从物理世界产生，经由自有基础设施采集，在自有平台上聚合训练——这才是真正的独占。纯 SaaS 的数据几乎都属于客户。

投资逻辑从「找最好的 SaaS 产品」变成了「找成功转型为数据基础设施的公司」。

这里有一个悖论：按第一节的定义，价值锚在数据而非软件的公司，本就不算纯 SaaS。所以这篇文章真正的结论是——活下来的，已经不是纯 SaaS 了。它们以订阅制交付，但核心资产是不可替代的数据，不是标准化的软件功能。

对每家公司问三个问题：数据越多产品越好吗？数据只能从你这里出吗？Agent 来了还能收到钱吗？能同时回答「是」的公司屈指可数——但它们是 AI 时代真正的赢家。

如果答案是 「把数据变得可用」——你是黄页。
如果答案是 「拥有别人拿不到的数据」——你比以前更值钱。
如果答案是 「监管要求必须经过我」——你很安全，但要警惕临界点。
如果答案是 「大家都用我」——要看「大家」是人还是 Agent。
