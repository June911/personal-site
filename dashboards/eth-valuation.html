方法论

框架来源：IOSG Ventures 2026-02 Weekly Brief #313《以太坊再定价：从 Rollup-Centric 到"安全性结算层"》(Jacob Zhao & Jiawei)。复用 Hashed ethval.com 底层模型，在估值架构与权重逻辑上重塑：结构映射至四大价值象限，显著上调安全性与结算溢价权重，弱化协议收入与 L2 扩张贡献。

核心主张：ETH 的定价正从现金流模型转向资产溢价模型。"安全性结算层 + 原生货币属性"是价值中枢，平台效应与收入分别是牛市放大器和熊市安全垫。

四象限逻辑：
1. 安全结算层 (45%) — Validator Economics (收益均衡) + Staking DCF (永续折现)，刻画全球安全结算层的制度性溢价。避险期权重上调。
2. 货币属性 (35%) — 分层 MV=PQ (高频结算/中频金融/低频抵押)，量化 ETH 在三个货币功能的结构性需求。
3. 平台效应 (10%) — 信任修正 Metcalfe: a×users^b + m×Σ(L2_TVL×trust)。增长期权，牛市放大器。
4. 收入底线 (10%) — min(P/S floor, Fee Yield floor)。现金流地板，熊市托底。

局限性
  1. 循环引用：Validator Economics 和 Staking DCF 均用 current_price 计算收入。原文明确指出：含现价输入的模型不作为公允价值锚，仅作仓位与风险偏好调节指标
  2. 安全性溢价不可证伪："安全结算层"价值本质是制度性叙事溢价，无法被链上数据否决
  3. 活跃地址为默认值：Metcalfe 项需要日活跃地址数据，当前使用 500K 默认值（CoinGecko 免费 API 不提供）。接入 Etherscan/Dune 可改善
  4. 参数主观性高：discount_rate、velocity、trust_score、Metcalfe 系数等均为主观选择，敏感度高。请参考情景分析 tab 的敏感度矩阵评估影响
  5. 动态校准简化：原文的 Regime Adaptation 包含宏观资金成本、市场相对强弱、链上拥挤度三层约束。当前仅用 24h 价格变动粗略判定，远未达到原文精度
  6. 快照时刻：所有数据为运行时点值，不含时间序列趋势分析
